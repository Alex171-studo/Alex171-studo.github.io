# Kobold

Difficulté : Easy

OS: Linux

Plateforme: HTB Saison 10

### Summary

`Kobold` est une machine `hackthebox` focalisée sur l’exploitation d’un site web qui utilise une version vulnérable de `MCPJam`  puis l’escalade de privilège via `docker`.

### Host Resolution

Nous allons commencer par associer l’adresse ip de notre cible au nom de domaine `kobold.htb`

```bash
echo "10.129.27.99 kobold.htb" >> /etc/hosts
```

### Nmap Scan

La reconnaissance a débuté par un scan `nmap` de la cible afin de déterminer les ports ouverts

```bash
nmap -sC -sV -p- kobold.htb
```

On a obtenu

```bash
PORT   STATE SERVICE VERSION
80/tcp   open  http      nginx 1.24.0 (Ubuntu)
|_http-title: Did not follow redirect to https://kobold.htb/
|_http-server-header: nginx/1.24.0 (Ubuntu)
3552/tcp open  taserver?
| fingerprint-strings: 
Service Info: Host: localhost; OS: Linux; CPE: cpe:/o:linux:linux_kernel
```

Le scan révèle la présence de deux ports :

- Port HTTP (80)
- Port 3552, un port non standard

### **Web application Discovery**

Les deux ports sont basés sur TCP donc ils hébergent surement des pages web. Commençons l’exploration par le port 80

```
https://kobold.htb
```

![image.png](attachment:c4797133-f32e-4d4a-baff-e0a34cb7818e:image.png)

La page d’accueil présente une entreprise nommée Kobold Systems qui développe une plateforme logicielle de gestion des opérations informatiques (gestion des services et des conteneurs, automatisation & agents intelligents,etc)

Explorons le service qui tourne sur le port `3552` en nous rendant à l’adresse

```
https://kobold.htb:3552
```

On est redirigé sur une page de login

![image.png](attachment:f4b060db-2c24-4263-a11f-7f3dea2b0c7b:image.png)

Il s’agit de `Arcane` un service de gestion des conteneurs ce qui pourrait indiquer que sur la machine tourne un conteneur. 

### Subdomains enumeration

Nous allons énumérer les sous-domaines associé au domaine `kobold.htb` 

```bash
gobuster vhost -u https://kobold.htb -w /usr/share/wordlists/seclists/Discovery/DNS/subdomains-top1million-5000.txt --append-domain -k
```

On a le résultat

```
10.129.27.99 kobold.htb mcp.kobold.htb
```

Nous allons ensuite explorer ce sous-domaine en nous rendant à l’adresse 

```
https://mcp.kobold.htb
```
